## 一、背景与目标

### 1.1 背景

当前系统的业务基础数据（SKU、仓库、渠道、组织、日历、关系配置）全部依赖 Alibaba DataQ 平台写入 `cdop_biz` schema 的只读表，本系统无法主动维护这些数据，存在以下问题：

- DataQ 平台数据同步存在延迟，影响业务计划的实时性
- 数据质量问题（错误编码、缺失数据）无法在本系统内修正
- 渠道、品类数据仅有编码集合查询，缺乏完整树形结构管理能力
- 关系数据（仓库-SKU、SKU-经销商等）通过硬编码过滤，维护成本高

### 1.2 目标

新建 `cdop_master` schema，建立本系统自主管理的业务基础数据层：

1. 提供完整的基础数据 CRUD 管理界面
2. 支持 Excel 批量导入，降低数据维护成本
3. 替换现有对 DataQ 只读表的直接依赖
4. 为需求预测、库存分配等业务模块提供稳定的基础数据服务

### 1.3 用户群体

| 用户角色 | 主要诉求 |
|---------|---------|
| 系统管理员 | 初始化数据导入、日常数据质量修正 |
| 供应链计划员 | 查询基础数据，确认数据准确性 |
| 数据维护专员 | 日常基础数据的增删改维护 |

## 二、功能范围

### 2.1 模块结构

```
基础数据管理
├── 商品管理
│   ├── SKU管理
│   └── 品类管理
├── 仓库管理
│   ├── 仓库管理
│   └── 工厂管理
├── 渠道管理
│   ├── 渠道管理
│   ├── 经销商管理
│   └── 经销商地址管理（内嵌于经销商详情）
├── 组织管理
│   └── 组织机构管理
├── 日历管理
│   └── 业务日历
└── 关系配置
    ├── 仓库-SKU关系
    ├── SKU-经销商关系
    ├── 组织-经销商关系
    └── 产品-销售SKU转换关系
```

### 2.2 数据库规划

| Schema | 用途 |
|--------|------|
| `cdop_master` | 本模块新建，本系统完整 CRUD |
| `cdop_biz` | DataQ 只读表，保留，不再写入 |
| `cdop_sys` | 系统管理表，不变 |

### 2.3 开发优先级

| 优先级 | 功能模块 | 说明 |
|--------|---------|------|
| P0 | SKU管理、品类管理、仓库管理、工厂管理、渠道管理、经销商管理 | 其他所有业务功能的前置依赖，必须最先完成 |
| P1 | 仓库-SKU关系、SKU-经销商关系、组织-经销商关系 | 需求计划和库存分配的核心关系数据 |
| P2 | 组织机构管理、日历管理、产品-销售SKU转换关系、经销商地址管理 | 相对独立，可在 P0/P1 完成后实施 |

## 三、通用规范

以下规范适用于本模块所有子功能。

### 3.1 列表页标准功能

| 功能 | 说明 |
|------|------|
| 条件筛选 | 关键编码/名称模糊搜索 + 状态/类型等精确过滤，支持重置 |
| 分页查询 | 默认每页 20 条，支持切换 10/20/50/100 |
| 新增 | 弹窗表单，提交后刷新列表 |
| 编辑 | 弹窗表单，回显当前数据 |
| 删除 | 单条删除（行内按钮）+ 批量删除（勾选后操作）；逻辑删除（status=0），不物理删除 |
| Excel 导入 | 弹窗上传，提供模板下载链接 |
| Excel 导出 | 按当前筛选条件导出，文件名含日期时间戳 |
| 批量启用/停用 | 勾选后操作，操作前弹确认提示 |

### 3.2 表单校验规范

| 场景 | 规则 |
|------|------|
| 编码字段 | 新增时校验唯一性（接口查重），编辑时编码不可修改 |
| 必填字段 | 前端实时校验，提交前拦截 |
| 日期有效期 | 开始日期不得晚于结束日期 |
| 关联编码 | 关系表中的编码需校验在主数据表中存在 |

### 3.3 Excel 导入规范

| 规则 | 说明 |
|------|------|
| 文件格式 | 仅支持 `.xlsx` |
| 文件大小 | 不超过 10MB |
| 最大行数 | 单次导入不超过 10000 行 |
| 错误处理 | 逐行校验，成功行写入，失败行汇总返回错误报告（行号 + 错误原因） |
| 名称回填 | 导入时只需填写编码，系统自动从主数据表回填对应名称字段 |
| 重复处理 | 编码已存在时，默认**更新**（不新增），可在导入弹窗中配置"跳过"或"覆盖" |

### 3.4 删除保护规则

| 场景 | 规则 |
|------|------|
| 品类有子节点 | 不允许删除，提示"请先删除下级品类" |
| 渠道有子节点 | 不允许删除，提示"请先删除下级渠道" |
| 渠道被经销商引用 | 不允许删除，提示"该渠道已有经销商绑定" |
| 仓库/SKU/经销商被关系表引用 | 给出提示，需用户二次确认是否强制删除（同步清理关系数据） |

- 生命周期：下拉（在售/停售/新品）
- 保质期（天）：数字输入

- 重量单位

**价格：**
- 单价 / 价格单位

**品类（三级联动）：**
- 一级品类编码+名称（从品类树选择，自动填充 lv1_category_code 和 lv1_category_name）
- 二级品类（联动）
- 三级品类（联动）
- 财务分类编码+名称

**品牌：**
- 品牌编码 / 品牌名称 / 品牌集团 / 品牌组织

**其他：**
- 状态（有效/无效）
- 备注

#### 4.1.3 Excel 导入

**模板必填字段：**
- SKU编码、SKU名称

**可选字段：**
- 69码、生命周期、保质期、净重、毛重、重量单位、体积、体积单位、销售单位、计划单位系数、原子单位系数、单价、价格单位、是否赠品(0/1)、品牌编码、品牌名称、品牌集团、品牌组织、一级品类编码、二级品类编码、三级品类编码、财务分类编码、状态(0/1)、备注

**导入逻辑：**
1. 校验 sku_code 格式非空
2. 若 sku_code 已存在 → 覆盖更新
3. 若填写了品类编码 → 自动从 `mst_category_info` 查询并回填品类名称
4. 逐行写入，失败行不回滚成功行，返回错误报告

### 4.2 品类管理

**对应表：** `cdop_master.mst_category_info`

#### 4.2.1 页面布局

左树右表布局：
- **左侧：** 三级品类树，支持展开/收起，点击节点过滤右侧列表

| 删除节点 | 节点右键菜单 | 有子节点时不允许删除 |

#### 4.2.3 列表列

| 列名 | 说明 |
|------|------|
| 品类编码 | category_code |
| 品类名称 | category_name |
| 层级 | 一级/二级/三级 |
| 父级编码 | parent_code |
| 排序 | sort_order |
| 状态 | 有效/无效 |
| 操作 | 新增子节点、编辑、删除 |

#### 4.2.4 Excel 导入模板

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 品类编码 | 必填 | 唯一 |

## 九、关系配置

### 9.1 通用说明

关系配置类功能的特殊要求：
- Excel 批量导入为**必须功能**（关系数据量大，手动逐条录入不现实）
- 导入时只填编码，系统自动校验编码在主数据中存在并回填名称
- 所有关系表支持时间有效期（begin_date / end_date）
- 支持按"当前有效"快速过滤（当前日期 ≥ begin_date 且 ≤ end_date）
