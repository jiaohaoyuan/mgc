# 供应链决策平台 (SCMP) 技术栈详解手册

本文档旨在详细介绍本项目所采用的技术体系。通过阅读本文档，您可以了解项目底层使用的各项技术工具、它们的通用用途，以及在本项目中的具体应用场景。

## 1. 核心技术概览 (Core Tech Stack)

本项目采用现代化的**前后端分离**架构，前端基于 Vue 生态构建，后端基于 Node.js 异步非阻塞环境，配合高性能关系型数据库与内存缓存。

| 维度 | 技术选型 |
| :--- | :--- |
| **前端框架** | Vue 3 (Composition API) |
| **构建工具** | Vite + TypeScript |
| **UI 组件库** | Element Plus |
| **状态/路由** | Pinia + Vue Router |
| **后端运行环境** | Node.js (LTS �汾) |
| **Web 开发框架** | Express (Node.js 框架) |
| **数据库** | MySQL 8.x / MariaDB |
| **内存/缓存** | Redis |
| **可视化/数据** | ECharts + SheetJS (XLSX) |

## 2. 前端技术体系

### 2.1 Vue 3 (Composition API)
- **技术介绍**：Vue.js 是一套用于构建用户界面的渐进式 JavaScript 框架。Vue 3 引入了组合式 API (Composition API)，显著提升了逻辑复用性、类型推断和大型项目中的代码组织效率。
- **在本项目中的作用**：
    - **核心底盘**：所有的业务页面（如 `MdmSkuList.vue`, `IntelligentOrdering.vue`）均基于 Vue 3 开发。
    - **响应式驱动**：利用 Vue 的响应式系统，实现复杂的供应商与 SKU 联动逻辑，确保数据变化时 UI 自动更新。
    - **Vapor 模式/Beta 实验**：项目尝试采用了 Vue 3 Beta/Vapor 相关特性，追求极致的初次渲染速度。

### 2.2 Vite & TypeScript
- **技术介绍**：
    - **Vite**：下一代前端构建工具，利用浏览器原生 ES 模块实现极速热更新（HMR）。
    - **TypeScript**：JavaScript 的一个超集，添加了静态类型校验。
- **在本项目中的作用**：
    - **极速开发体验**：显著缩短了开发环境的启动时间，在修改大型组件（如 ECharts 画板）时能实现几乎无感的瞬间更新。
    - **工程稳固性**：在前端逻辑层大量使用类型定义，有效规避了在处理 SKU 复杂 JSON 数据结构时的“未定义”错误。

### 2.3 Element Plus
- **技术介绍**：基于 Vue 3 的桌面端主流组件库，提供了一系列美观、高性能且符合现代化标准的 UI 控件。
- **在本项目中的作用**：
    - **标准化界面**：项目中所有的表格（Table）、表单（Form）、分页（Pagination）和弹窗（Dialog）均统一使用 Element Plus 提供的组件，确保了全平台视觉体验的一致性。
    - **交互增强**：利用其内置的 `Message` 和 `MessageBox` 实现操作反馈。

### 2.4 Pinia & Vue Router
- **技术介绍**：
    - **Pinia**：Vue 的官方状态管理库，简洁高效。
    - **Vue Router**：Vue.js 的官方路由管理器。
- **在本项目中的作用**：
    - **集中状态管理**：在 `stores/appStore.ts` 中统一存储用户信息、多选列表、全局菜单状态等。
    - **单页导航**：实现了整个项目无刷新的平滑页面跳转，并配合 Vue 的 `keep-alive` 实现了页面缓存。

## 3. 后端技术体系

### 3.1 Node.js & Express
- **技术介绍**：
    - **Node.js**：基于 Chrome V8 引擎的 JavaScript 运行时。
    - **Express**：一个简洁而灵活的 Node.js Web 应用框架。
- **在本项目中的作用**：
    - **API 服务中枢**：代码位于 `/server/index.js`，它充当了系统的“大脑”，负责接收前端的所有 HTTP 请求。
    - **高性能路由**：处理包括用户登录、SKU 主数据增删改、权限控制等在内的数百条 RESTful API。

### 3.2 Multer & XLSX (SheetJS)
- **技术介绍**：
    - **Multer**：Node.js 中间件，用于处理 `multipart/form-data` 类型的表单数据，主要用于文件上传。
    - **XLSX**：功能强大的 Excel 数据解析与导出库。
- **在本项目中的作用**：
    - **大通量导入管道**：在“基础数据管理”模块中，Multer 将前端上传的 Excel 文件读入内存，随后由 XLSX 库进行精细化解析，从而实现了数千行 SKU 数据的批量秒级导入。

### 3.3 JWT (JSON Web Token) & Bcryptjs
- **技术介绍**：
    - **JWT**：跨域身份验证的标准。
    - **Bcryptjs**：高性能的安全哈希算法库，用于密码加密。
- **在本项目中的作用**：
    - **核心安全屏障**：用户的密码在数据库中以 Bcrypt 强加密形式存储（不可逆）。用户登录后，服务器签发 JWT Token，作为后续所有私密 API 请求的身份令牌。

## 4. 数据库与中间件

### 4.1 MySQL (cdop_sys & cdop_master)
- **技术介绍**：广泛使用的开源关系型数据库管理系统。
- **在本项目中的作用**：
    - **持久化存储**：系统所有的静态与动态数据（部门架构、员工信息、SKU 档案、分销商关系）均持久化在 MySQL 中。
    - **多库分切方案**：项目将“系统配置（sys）”与“基础主数据（master）”在逻辑上物理分离，极大提高了系统的扩展性和后期数据中心迁移的便利性。

### 4.2 Redis
- **技术介绍**：开源的内存中数据结构存储系统，可用作数据库、缓存和消息中间件。
- **在本项目中的作用**：
    - **高频校验场**：本项目利用 Redis �洢 **短信验证码** (SMS Codes)。由于其内存读写特性，能瞬间响应身份校验请求，并利用过期机制 (TTL) 自动清理失效的验证码，减轻数据库压力并提高安全性。

## 5. 数据采集与可视化

### 5.1 ECharts
- **技术介绍**：由百度开源的一个功能强大的可视化图表库。
- **在本项目中的作用**：
    - **智慧大屏看板**：在 `IntelligentOrdering.vue` 等模块中，ECharts 负责将枯燥的供应链数字转化为动态、美观的地图分布图、折线图和智能演算仪表盘，赋予数据极高的决策价值。
