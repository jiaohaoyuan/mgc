# 页面业务逻辑与数据联动说明

更新时间：2026-04-13  
适用代码范围：`src/views/*`、`src/components/MdmGeneric*.vue`、`server/*.js`

## 1. 系统数据域（后端本地库）
- `system`：账号、部门、岗位、角色、权限（组织权限域）
- `master`：主数据（SKU、经销关系、仓库/工厂/渠道/组织、关系表、日历）
- `biz`：业务数据（订单、库存、调拨、渠道经营、牧场统计等）
- `platform`：平台数据（操作日志、安全日志、导入导出任务、通知、系统配置、归档作业、工作流等）

## 2. 页面业务逻辑清单（按模块）

## 3. 重点链路：创建订单后，哪些页面会同步更新

## 3.1 订单创建入口
- 当前“可创建订单”的页面是：`/intelligent-closed-loop`（订单闭环中心）
- 操作入口：页面按钮“新建订单”
- 对应接口：`POST /api/orders/phase2`
- 写入表：`biz.order_headers`、`biz.order_lines`，并写入 `platform.operation_logs`（CREATE）

## 3.2 创建后立即可见的页面变化
1. 订单闭环中心（同页）
- 列表新增订单号、客户、区域、来源、数量、金额、状态
- 看板指标更新：订单总量、分配成功率、异常率、履约率等
- 异常列表按规则重新计算（库存不足、效期风险、授权越界等）

2. 经营分析与管理驾驶舱 `/management-cockpit`
- `order-analysis`：订单量、金额、区域/渠道分布、履约与异常指标
- `overview`：总览指标联动变化
- `channel-analysis`：经销商排行与渠道结构会吸收新增订单事实

3. 渠道与经销商经营中心 `/channel-dealer-ops`
- `dashboard` 与 `analysis/overview`：销量、增长率、区域分布变化
- `authorizations/overreach-sales` 与风险扫描：新增订单可能触发“越权销售”风险

4. 操作日志与审计
- `/operation-log` 与 `/platform/audit-log` 可看到“新增订单”操作记录和快照差异

5. 平台监控 `/platform/monitor`
- 本次订单创建 API 调用会纳入接口调用量、时延、错误率统计

## 3.3 订单后续动作带来的跨页同步
1. 提交审核 `POST /orders/phase2/:orderNo/submit`
- 订单状态变为待审，审计轨迹新增 SUBMIT
- 订单闭环看板/列表、驾驶舱订单状态统计更新

2. 审核通过/驳回 `POST /orders/phase2/:orderNo/audit`
- 通过：进入待分配
- 驳回：回到驳回态，并释放已锁库存（若存在）
- 影响：订单闭环 + 库存运营（锁释放）+ 驾驶舱

3. 自动/人工分配 `POST /orders/phase2/:orderNo/allocate/*`
- 触发库存锁定同步：`synchronizeOrderLocks`
- 写入并更新：
  - `biz.inventory_locks`
  - `biz.inventory_ledger`（available/locked 数量变化）
  - `biz.inventory_transactions`（FREEZE/UNFREEZE）
  - `biz.inventory_warnings`
- 影响页面：
  - 订单闭环（已分配量、分配方案版本、异常）
  - 库存运营（台账、交易、锁定、预警）
  - 驾驶舱库存健康分与异常计数

4. 生成补货建议/补货决策
- 写入 `biz.replenishment_suggestions`
- 影响订单闭环详情与异常处理视图

5. 履约状态流转/关闭 `POST /orders/phase2/:orderNo/fulfillment`
- 关闭时释放订单锁（`releaseOrderLocks`）
- 影响订单闭环、库存运营、驾驶舱履约率

## 3.4 当前实现中的边界
- `/intelligent`（智能订购中心）当前为旧链路分析页，只读，不是新订单创建入口。
- 订单闭环动作当前不会自动生成工作流待办（`workflow_center`），工作流中心更多是独立任务/审批域与导入导出任务汇总。

## 4. 其他高频联动链路

## 4.1 主数据变更 -> 业务页面
1. 在 MDM 页面或治理平台生效后，`master` 主表被更新
2. 下游联动：
- 订单闭环 `config`（客户/SKU/仓库候选）与提交校验规则立即受影响
- 渠道经营（档案/授权/价格）补种或校验逻辑受影响
- 库存运营候选项与仓能力口径受影响
- 驾驶舱 MDM 质量与全局指标受影响

## 4.2 导入导出任务 -> 任务中心/通知/工作流
1. 主数据导入 `POST /master/import`：写入 `platform.import_tasks` + 通知 + 操作日志
2. 主数据导出 `GET /master/*/export`：写入 `platform.export_tasks` + 通知 + 操作日志
3. 联动页面：
- `/import-task`、`/export-task` 任务详情更新
- 顶栏未读数（`/notifications`）变化
- `/workflow-center` 的任务页会汇总显示 IMPORT/EXPORT 任务

## 4.3 角色权限变更 -> 可见菜单与接口权限
- 角色/权限变更后，`/me` 返回的 `permissionPaths` 会变化
- 前端侧边栏与路由守卫按 `permissionPath` 生效
- 精细化权限配置更新会记录安全日志与审计日志

## 5. 建议的使用口径（给业务/测试）
- “新建订单”统一在 `/intelligent-closed-loop` 做，不在 `/intelligent`。
- 联调验证时，订单链路建议按顺序验证：
  1. 新建 -> 2. 提交审核 -> 3. 审核通过 -> 4. 分配 -> 5. 履约状态流转
- 每一步同时核对 4 个联动面：
  1. 订单闭环数据是否变化
  2. 库存运营的锁定/台账/预警是否对应
  3. 驾驶舱指标是否刷新
  4. 审计/监控是否留痕
