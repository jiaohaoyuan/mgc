# 第一期前端联调回归清单

更新时间：2026-04-13  
适用范围：第一期“订单/库存口径统一 + 导入任务纳管 + 驾驶舱/渠道默认有效状态统计”

## 1. 联调前准备

1. 启动后端：`cd server && npm start`
2. 启动前端：`npm run dev`
3. 使用有业务权限账号登录，确认可访问：
- `/intelligent`
- `/intelligent-closed-loop`
- `/inventory-ops`
- `/import-task`
- `/management-cockpit`
- `/channel-dealer-ops`

## 2. 关键回归用例

### 2.1 旧智能订购页口径兼容（`/intelligent`）

1. 打开页面，确认订单列表可加载。
2. 核对状态：存在 `Pending`、`Matched`。
3. 触发分页切换，列表持续可用。

期望：
- 不再依赖旧 `biz.orders`，但页面行为保持兼容。

### 2.2 订单闭环库存口径（`/intelligent-closed-loop`）

1. 创建订单并提交审核。
2. 审核通过后执行自动分配。
3. 打开订单详情，查看分配结果与异常信息。

期望：
- 分配推荐/异常判断基于 `inventory_ledger`。
- 与库存运营页展示口径一致。

### 2.3 库存联动校验（`/inventory-ops`）

1. 在订单分配后查看：`库存台账`、`锁定`、`交易`。
2. 审核驳回或订单关闭后再次查看。

期望：
- 锁定/释放与订单动作一致。
- 台账 `available/locked` 与订单状态联动。

### 2.4 订单导入任务纳管（`/intelligent-closed-loop` + `/import-task`）

1. 在订单闭环执行导入（含至少1条错误数据）。
2. 打开导入任务中心，按业务类型筛选 `ORDER_PHASE2`。
3. 查看任务详情与失败明细下载。

期望：
- 生成 `IMPORT_TASK` 记录。
- 任务详情可见失败原因（`errors` 或 `failed_rows`）。
- 可下载失败CSV。

### 2.5 驾驶舱默认有效状态口径（`/management-cockpit`）

1. 对比草稿单/驳回单前后，观察订单分析与渠道分析统计。
2. 调用接口带参数验证：
- 默认（不传）
- `includeAllStatuses=true`
- `orderStatuses=APPROVED,CLOSED`

期望：
- 默认排除 `DRAFT/REJECTED`。
- 参数可覆盖默认过滤。

### 2.6 渠道经营默认有效状态口径（`/channel-dealer-ops`）

1. 打开看板与分析页，核对销量、增长、风险统计。
2. 与订单闭环当日新增单对照。

期望：
- 默认统计口径为有效订单。
- 看板与分析页数据趋势一致。
