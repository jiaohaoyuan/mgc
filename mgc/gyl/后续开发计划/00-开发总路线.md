# 供应链决策平台后续开发总路线

## 1. 文档目的
本目录用于沉淀项目后续开发路线，目标是让当前系统从“可运行原型”逐步升级为“企业级供应链运营平台”。

本套文档遵循以下原则：
- 每个阶段都是可独立推进的开发主题。
- 每个阶段都拆到可继续细化为数据库、接口、页面和流程的粒度。
- 后续你可以直接基于某一份文档继续让我输出更细的设计或直接开始编码。

## 2. 当前项目现状判断
结合现有代码，系统已经具备以下基础：
- 平台基础能力：登录、找回密码、用户、角色、岗位、部门、权限、路由鉴权。
- 主数据能力：SKU、经销商关系、仓库、工厂、渠道、组织、业务日历、关系表管理。
- 数据交换能力：Excel 导入、导出、通用 CRUD、分页查询。
- 业务展示能力：牧场概览、智能订购中心、订单分析、地图与图表展示。
- 后端基础能力：JWT 鉴权、接口权限校验、本地 JSON 持久化、通用数据接口。

当前缺少的关键能力：
- 订单执行闭环。
- 库存与仓配执行。
- 审批与流程驱动。
- 主数据治理与审计追溯。
- 统一待办、通知、任务体系。
- 经营分析与管理驾驶舱。
- 企业级运维、安全与平台能力。

## 4. 推荐实施方式
建议每个阶段按以下顺序继续细化和开发：
1. 明确业务目标和使用角色。
2. 拆分数据库实体与状态流转。
3. 定义接口清单与字段。
4. 定义页面结构与交互动作。
5. 定义权限点、审批点、日志点。
6. 再开始前后端编码。

## 5. 推荐优先级
如果只能先做最核心的三块，建议优先：
1. 平台基础加固。
2. 订单与智能订购闭环。
3. 主数据治理平台。

原因：
- 平台基础加固会成为所有新模块复用的公共能力。
- 订单闭环直接决定系统业务价值。
- 主数据治理决定后续业务数据是否可靠。
