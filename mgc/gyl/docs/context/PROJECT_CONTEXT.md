# PROJECT_CONTEXT

## 这个项目是什么

这是一个乳制品供应链决策平台，业务背景围绕“认养一头牛”式的奶制品供应链展开。

它不是单一模块系统，而是一个把主数据、组织权限、业务运营和分析看板放在一起的综合平台。

## 当前项目的业务范围

系统主要包含以下业务域：

- 组织、用户、角色、岗位、权限
- 字典、导入导出、日志、平台运维
- 奶制品主数据管理
- 智能订货与订单闭环
- 库存与仓配运营
- 渠道与经销商经营
- 流程与待办
- 经营分析与管理驾驶舱
- 牧场与奶源运营

## 奶制品主数据的关键事实

当前仓库已经把品类体系收敛为乳制品专用结构。

一级分类核心方向是：

- 液态奶制品
- 固态奶制品
- 半固态奶制品

SKU / SPU / 品类规则已经有专门的后端规则和校验文件维护，不能只改页面展示，不看底层规则。

## 技术结构

### 前端

- 目录：`src/`
- 形态：Vue 单页应用
- 关键点：
  - `src/App.vue` 负责应用外壳、导航、部分全局交互
  - `src/router/index.ts` 负责路由和权限守卫
  - `src/stores/appStore.ts` 负责权限上下文和系统基础数据
  - `src/views/` 是页面级业务模块
  - `src/components/` 有 MDM 通用组件

### 后端

- 目录：`server/`
- 形态：Express API 服务
- 关键点：
  - `server/index.js` 是主入口和大部分 API 注册点
  - `server/*.js` 包含业务域模块、种子数据、规则、治理逻辑
  - `server/localDb.js` 负责本地 JSON 数据的加载与落盘

### 数据持久化

- 活跃数据文件：`server/local-data/db.json`
- 这意味着很多修改不是只影响代码，也会直接影响本地业务数据形态

## 认证与权限

- 前端通过本地 token 驱动登录态
- Axios 请求会自动带认证头
- 路由层有权限守卫
- 后端层也按 API 路径做权限拦截
- MDM 相关页面里有超管专属能力

## 当前代码库已知的重要实现方向

- 页面调用应优先走统一 API 基址，不要硬编码 `localhost`
- 页面之间的数据联通性已经按 API 路由检查过
- 奶制品品类、SKU、SPU 之间已有一轮迁移和校验逻辑
- 导入上传对文件类型已有一定限制
- 部分业务页面懒加载，首次跳转体感会受 chunk 体积影响

## 修改时最容易忽略的链路

一个页面改动，常见不是只改页面本身，还会连到：

- 路由权限
- store 中的基础数据
- API 返回结构
- 本地 JSON 数据结构
- 主数据规则
- 导入导出模板
- 治理/校验逻辑

## 什么叫“改对了”

在这个仓库里，一个修改算真正完成，至少要满足：

- 页面行为符合需求
- API 没断
- 权限没串
- SKU / SPU / 品类规则没被破坏
- 构建通过
- 如果涉及后端数据规则，校验测试通过
